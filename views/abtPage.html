About

It's multiplayer Risk™! With random country generation! Or at least it will be eventually!

Usage

For now, just look at the pretty countries. Soon there'll actually be stuff to do.

Server

The server runs on a full JavaScript stack. This includes MongoDB as the database, Express.js as the server software, Angular (1.x) for the front end, and Node.js for the back end. By their powers combined, they form Captain MEAN-stack.

Technical Stuff

Conkr uses the following technologies:

  • AngularJS for front-end responsiveness.
  • NodeJS and ExpressJS on the back-end for the server.
  • Websockets, via socket.io, to allow quick communication between the server, phones, and game. One major advantage to websockets is that both the server and the client can initiate a conversation, whereas with normal HTTP (without long-polling), the server can never send information to the client without the client first requesting that information.
  • HTML5's Canvas.
  • The map itself is generated with something called a Voronoi Diagram. As I'm not really a math dude, I'm not 100% sure how this works. However, generally speaking, it generates a number of random points ('sites') on the canvas. It then draws an edge equidistant between each pair of points. Finally, there's some Math, and maybe even a bit of Science, and you get a map.
  • MongoDB for storing currently in-progress games. I may consider switching this to MySQL later (via Sequelize), depending on the conversion difficulty/time.
